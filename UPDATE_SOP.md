# 商业化竞对洞察平台 · 更新 SOP

> **每次新 session 开始时，AI 必须先读完本文件，再执行任何操作。**
> 本文件是强制 checklist，不得跳过任何阶段。

---

## 🔴 核心禁令（任何情况下不得违反）

1. **未经用户明确确认内容草案，绝对不能动 `index.html`**
2. **HTML 改动前，必须完成 Phase 1 → Phase 2 → Phase 3 三个阶段的用户确认**
3. **不得自行决定某篇文章的日期，必须从原文提取或向用户确认**
4. **不得跳过 Phase 2 草案确认，哪怕内容"看起来很明显"**

---

## 📋 五阶段流程

---

### Phase 0：Session 启动检查

**AI 在收到更新任务后，立即执行以下检查：**

- [ ] 读取本 SOP 文件（`UPDATE_SOP.md`）
- [ ] 读取 `index.html` 的 timeline 部分，确认当前最新期号（active panel 是哪一期）
- [ ] 向用户说明：当前最新期是 XX，本次将新建 YY 期
- [ ] 确认用户提供的信源列表（钛媒体 URL / 微信文章 URL / 其他）

**⛔ 卡点：** 用户没有提供信源，不得自行开始采集。等待用户说"开始"或提供 URL。

---

### Phase 1：内容采集与核实

**对每一篇文章，必须完成：**

- [ ] 抓取原文，提取标题、发布日期、核心内容（200字以内摘要）
- [ ] **日期必须从原文提取**，不得猜测或使用文件名/URL中的日期
- [ ] 如果原文日期不在目标周期内（通常为上周四至本周三），标注日期异常
- [ ] 微信文章：用户提供 URL → 立即抓取 → 提取日期和摘要，**不得预设内容**

**⛔ 卡点：** 某篇文章无法访问或日期存疑，必须告知用户，由用户决定是否替换，不得自行选择备选文章。

---

### Phase 2：草案确认（🔴 最关键卡点）

**以表格形式展示完整草案，包含：**

| # | 模块 | 来源 | 发布日期 | 标题/要点摘要 |
|---|------|------|----------|--------------|
| 1 | 竞对·腾讯 | ... | YYYY-MM-DD | ... |
| ... | ... | ... | ... | ... |

**草案展示完毕后，AI 说：**
> "以上是本期 [日期] 的草案，共 X 条。请确认是否可以继续，或告知需要调整的地方。"

**⛔ 卡点：** 用户没有明确说"可以"/"确认"/"没问题"之前，**绝对不能**：
- 开始写洞察文字
- 开始写任何 HTML
- 开始修改 timeline 按钮

**常见错误（已发生过）：**
- ❌ 用户还没确认，AI 就说"好的我来写HTML"
- ❌ 用户确认了内容，AI 却先动了 timeline 按钮
- ❌ 草案中文章日期是几个月前的老文章

---

### Phase 3：今日洞察确认

**用户确认草案后，AI 询问：**
> "请问今日洞察从哪几条来写？（建议选 2-3 条最有战略价值的）"

**用户指定来源后，AI 展示洞察草稿（纯文字，不含 HTML），格式为：**

```
洞察1标题：xxx
摘要：xxx（3-4句话，含竞对动向 + 行业变革 + 对快手启示）

洞察2标题：xxx
...
```

**⛔ 卡点：** 用户对洞察内容没有异议（或明确说"可以"）后，才进入 Phase 4。

---

### Phase 4：HTML 编写与插入

**只有完成 Phase 2 + Phase 3 的双重确认后，才能执行以下操作：**

**操作顺序（必须按顺序）：**

1. **先改 timeline 按钮**：在现有 active 按钮之前插入新按钮，并将旧 active 改为普通按钮
2. **再写新 panel HTML**：在 `panel-[上一期]` 之前插入 `panel-[新期]`，包含：
   - 今日洞察块（`date-panel active`，今日洞察 + 核心竞对 + 重点行业）
   - 将旧的 `date-panel active` 改为 `date-panel`（去掉 active）
3. **HTML 写完后自检**：
   - [ ] 新 panel id 格式是否正确（`panel-YYYYMMDD`）
   - [ ] 所有文章链接是否使用了真实 URL
   - [ ] 日期标注是否与草案一致
   - [ ] `<details>` 折叠块是否完整闭合

**⛔ 卡点：** HTML 写完后，向用户报告："HTML 已插入，请刷新页面确认显示效果。"等用户确认无误再执行 Phase 5。

---

### Phase 5：发布

**用户确认页面效果后，按顺序执行：**

1. `git add index.html`
2. `git commit -m "feat: 新增 panel-[日期]（竞对洞察·X月X日）\n\n[简要说明本期核心内容]"`
3. `git push`（如有冲突先 `git pull --rebase` 再 push）
4. 询问用户：**"是否需要推送 KIM 消息？"**
5. 用户确认后：`python3 scripts/kim_push.py`

**⛔ 卡点：** git push 成功后，向用户确认是否推 KIM，不得自动推送。
