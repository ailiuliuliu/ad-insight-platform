# 商业化洞察平台更新 SOP

## 适用范围
用于 `ad-insight-demo` 的日常内容更新、HTML 写入、网站部署与 KIM 推送。

---

## 一、执行原则

### 1. 两层门槛必须同时检查
每次更新都必须同时检查：

- **信息源数量门槛**：公众号、行业媒体是否达到要求
- **模块覆盖门槛**：字节、小红书、腾讯、百度、游戏与内容、本地生活、电商广告是否都补齐

只要任一门槛未达标，就不能视为完成。

### 2. 没达标时默认继续补齐
如果门槛未满足，应默认继续搜索、筛选、补齐，不需要等待用户再次明确说“继续”。

但有一个前提：
- **在真正写入 HTML、部署或推送前，仍然必须先把模块化草案给用户确认。**

### 3. 单条新闻只能归属一个模块
禁止一稿多投。

如果一条新闻既能解释竞对平台动作，又能解释行业赛道趋势：
- 必须二选一
- 按“最核心价值”归属到唯一模块

后续判断缺口时，也必须基于这个“单一归属”结果重算，不能按总条数凑数。

### 4. 缺 n 条时，四家媒体各至少搜 n 条
在单一归属规则下，如果还缺 `n` 条，必须对以下四家媒体：

- 36氪
- 钛媒体
- 虎嗅
- 21财经

**各至少搜索 `n` 条候选内容**，再从总候选池中选出最有价值的 `n` 条补入缺口模块。

禁止只在单一媒体上搜两三条就宣布补齐。

### 5. 内容质量标准高于凑数
需要主动删除：

- 低价值方法论稿
- 弱运营稿
- 弱榜单稿
- 明显时效性不足的旧稿

时效性要求：
- 优先近一周内容
- 其次近两周内容
- 即使勉强还在原时间窗口内，若讨论价值明显下降，也应主动剔除

### 6. 竞对模块优先于赛道模块的判断条件
如果一条内容虽然属于某个行业，但其核心价值更在于体现某个平台的：

- 方法论升级
- 产品能力变化
- 商业化动作
- 经营策略升级

则应优先归入**竞对模块**，而不是赛道模块。

### 7. 今日洞察必须从最终确认内容中提炼
流程必须是：

1. 先补齐模块内容
2. 再完成唯一归属
3. 再给用户确认最终模块清单
4. 最后从最终清单中选最重要的 3 条提炼“今日洞察”

不允许脱离最终模块池，单独生成一套洞察口径。

### 8. 页面展示规则
页面仍然保持**时间轴形式**，新一期内容通过新增 timeline 节点进入。

同时，自本次起：
- 页面文案 `历史记录` → `Timeline`

## 二、标准执行流程

### Step 1：收集候选信息
- 收集公众号候选
- 收集行业媒体候选
- 初步去重
- 标记时间、来源、主题、潜在归属模块

### Step 2：按唯一归属重排候选池
- 所有候选只允许进入一个模块
- 按“核心价值”分配到竞对模块或赛道模块
- 基于唯一归属结果重算各模块缺口

### Step 3：按缺口补搜行业媒体
如果仍有缺口：
- 对 36氪、钛媒体、虎嗅、21财经分别补搜
- 若还缺 `n` 条，则四家媒体各至少搜 `n` 条候选
- 从总候选池中筛选价值最高的内容补入

### Step 4：做内容质量筛选
逐条检查：
- 是否足够新
- 是否信息密度高
- 是否不是弱方法论 / 弱运营 / 弱榜单
- 是否有明确平台动作、商业化价值或赛道趋势价值

### Step 5：输出模块草案给用户确认
在写 HTML 前，必须先给用户展示：
- 各模块最终条目
- 每条内容的归属模块
- 今日洞察候选（如用户需要）

### Step 6：写入 HTML
确认后再改 `index.html`：
- 保持时间轴结构
- 新增当期日期 panel
- 更新 timeline 节点
- 如有要求，同步更新文案（例如 `Timeline`）

### Step 7：本地验证
至少检查：
- 时间轴按钮是否正确
- panel 切换是否正确
- 新内容是否显示完整
- 关键链接是否合理
- 页面结构无明显异常

### Step 8：部署到线上
推荐流程：
1. `git add` 指定文件
2. `git commit`
3. `git pull --rebase`（如远端有更新）
4. `git push origin main`
5. 等待 GitHub Pages 部署

### Step 9：确认线上生效
检查线上页面是否出现：
- 新日期节点
- 新增模块内容
- 本次要求的页面文案修改
- 关键替换条目是否在线可见

### Step 10：如需要，执行 KIM 推送
- 从最新 `index.html` 提取 3 条洞察
- 推送前确认脚本可运行
- 推送成功后记录结果

## 五、Git 提交流程建议

### 推荐提交范围
优先只提交本次真正相关的文件，例如：
- `index.html`
- `UPDATE_GUIDE.md`
- `scripts/kim_push.py`

### 不建议混入提交的文件
除非用户明确要求，否则不要把以下文件顺手带上：
- `scripts/kim_push.log`
- `scripts/.last_push_date`
- 其他运行时日志、缓存、临时文件
